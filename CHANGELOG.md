# Changelog

All notable changes to Moongate, newest first. Versions follow `vMAJOR.MINOR.PATCH+BUILD` where `BUILD` is a monotonic integer used by the in-app update banner.

| Version | Changes |
|---|---|
| **v0.9.52** | **go2rtc / WebRTC webcams now display.** A camera configured through **go2rtc** (a `stream.html` URL in Mainsail's webcam settings, e.g. WebRTC mode) showed nothing anywhere in the app - not the dashboard tile, not the full-screen camera, not even the embedded Mainsail page - while working fine in a desktop browser. Three fixes: **(1)** the app recognises a go2rtc player URL (auto-detected from Mainsail's webcam config, or pasted into a tile's gear as a custom URL) and swaps it for go2rtc's single-frame endpoint (`.../api/frame.jpeg?src=NAME`), so the tile and the full-screen camera render it like any other webcam - reverse-proxy subpaths and explicit ports survive the rewrite; **(2)** the embedded Mainsail/Fluidd page now allows **inline auto-playing video** (Android WebViews refuse scripted playback without a user gesture by default, and iOS forced fullscreen-only playback), so Mainsail's own go2rtc/WebRTC camera card finally plays in-app, on **both Android and iPhone**; **(3)** any camera URL that answers 200 with something that isn't an image (an HTML player page, an error page) no longer paints a broken image over the tile - the placeholder stays until real frames arrive. Honest limitation: a camera on a **public hostname** still can't ride the cloud tunnel remotely - the Pi's `/mg-extcam` relay deliberately forwards only to private home-network addresses (an SSRF guard); home WiFi, **Direct (LAN/VPN)** and your own VPN all work. `webview_flutter_android` / `webview_flutter_wkwebview` become direct dependencies at the versions already shipped. App-only update |
| **v0.9.51** | **Run it your way - a cloud-free Direct (LAN/VPN) mode, switchable per printer, plus required printer names.** **(1) Direct (LAN/VPN) mode**: the Add Printer screen now opens with a choice - **Moongate cloud** (the classic zero-setup pairing, unchanged) or **Direct (LAN/VPN)**, the fully **cloud-free** path built on the LAN-only plugin work in #201/#207 (thanks @packerlschupfer): install the Pi with `--lan-only`, then scan its `moongate://lan` QR or type its address, and the app talks **straight to Moonraker** over your network with **zero cloud calls, ever** - it works with no internet at all, and away from home you ride **your own VPN** (WireGuard, Tailscale). The tab states the honest trade-offs up front: no print notifications, remote access is your VPN's job, Moonraker's `trusted_clients` must cover the phone's network, and the Pi should keep a fixed address (the app stores it). **(2) Switch modes without re-pairing**: cloud-paired printers gain a **Direct (LAN/VPN) mode** switch in the printer page's edit dialog - both directions, effective on the next poll (the services read the flag live). Removing a Direct printer no longer fires a doomed cloud release call, and cloud-pairing a machine that already exists as a Direct tile **absorbs** the old tile, carrying its settings (and its name when the form is left untouched) onto the new one. **(3) Real names required**: the name field starts empty and its hint **cycles example names** ("Voron 2.4", "RatRig 300", "Wilma"...), and Pair/Add blocks with an inline error until a name is typed - the silent "My Printer" default is gone for good (the cloud database kept collecting rows literally named that). A Direct QR pre-fills the field with the Pi's hostname. **(4)** The **plugin-update badge** moved to the **bottom-left** of the camera box, leading the power button (was top-right). **(5)** Installer follow-ups from the #201 review: the cloudflared architecture check can no longer fail a LAN-only install on unsupported boards, and the LAN-only `MOONGATE_PAIR` console text points at the new Direct flow. The README gains a **"Run it your way"** cloud-vs-Direct comparison and a redrawn two-route **How it works** diagram. App-only update (plugin unchanged at 0.6.16). |
| **v0.9.50** | **The dashboard now flags out-of-date Pi plugins, and can update them with one tap.** Every `/status` poll already reports the printer's plugin version; the app now compares it against the plugin it shipped alongside (`kCurrentPluginVersion`) and shows a small **amber update badge** on that tile's camera box (compact tiles host it in the name row next to the power button) whenever the printer is behind. The badge is **deliberately persistent** - an out-of-date fleet is what let the 2026-06/07 quota spinners run - and clears itself the moment a poll reports the new version. Tapping it opens a dialog naming both versions: on **plugin 0.6.16+** (which advertises `plugin_can_self_update`) an **Update now** button fires the new `update_plugin` control action, which triggers Moonraker's own update manager on the Pi (`POST /machine/update/client?name=moongate`) and returns immediately - the update runs in the background and ends in a Moonraker restart, observed by the next polls. Older plugins can't be pushed (they predate the action), so the dialog shows the Mainsail Software-Updates route instead, and the one-tap path arrives with this very update. The action is **refused while a print is running or paused** (409; the dialog says to wait), and a pre-0.6.4 plugin that reports no version at all correctly reads as outdated. **Not app-only:** update the Moongate plugin (0.6.16) from Moonraker's update panel as well. |
| **v0.9.49** | **Pairing pop-up fixed and clearer, macro stars stick, and forgotten printers stop calling home.** **(1) The "How pairing works" pop-up stops re-prompting on every launch**: its "Don't show this again" tick has existed since v0.9.35 but was the last item inside the pop-up's scrolling area, below the fold on most phones, so users never saw it - it now sits pinned above the OK button, always visible. The plugin notice is also rewritten (all nine languages): install the plugin, then run the **MOONGATE_PAIR** macro (or type it in the Mainsail/Fluidd console) to get a pairing code or URL for QR-code pairing; the quick-start link stays underneath. **(2) Macro stars no longer vanish**: starring macros, closing the macro menu and reopening it dropped the new stars, and the next star toggle genuinely erased them (they only survived an app restart). The menu now reads the freshest star list every time it opens. **(3) Orphaned printers go dormant (plugin 0.6.15)**: a printer whose cloud registration is gone used to heartbeat every 5 minutes forever, wasting server quota (one such printer can burn most of the free tier); the plugin now goes quiet after a day of confirmed "no such printer" answers - or immediately when its owner removes it in the app, which now leaves a one-week server-side tombstone so the Pi gets a clear "released" answer - posts a notice in the Klipper console, and wakes instantly on MOONGATE_PAIR. **Not app-only:** update the Moongate plugin from Moonraker's update panel as well. Also: supabase_flutter 2.16.0, webview_flutter 4.14.1. |
| **v0.9.48** | **Hold-to-fullscreen camera, a Local-only switch, notification categories split, friendlier errors, and two runaway-client fixes.** **(1) Hold a tile's camera to open it full screen.** Press and hold anywhere on the camera square (with a haptic buzz) and the same pinch-to-zoom live view the corner eye icon opens appears, running at the camera's full Klipper-config frame rate; closing it returns the tiles to your chosen Raw/1s/3s/5s rate. In manual-reorder mode a long hold still drags the tile. **(2) Local-only switch** (user-requested): enable the **Local-only button** in the menu and a cloud toggle appears in the top bar next to the notifications pause button - tap it and Moongate stops using the remote **tunnel** entirely: status polling, webcams, the printer page and the background notification service all go **LAN-only** (zero remote traffic), printers not on your network settle to **offline**, and the printer page says "Remote access is switched off (Local only)" instead of spinning. The crossed cloud goes **orange** while active; the mode survives restarts, is deliberately **not** carried in backups, and switching the menu entry off also lifts the mode. **(3) Local feed rate defaults to Raw.** Fresh installs (and anyone who never chose a rate) now default the Wi-Fi dashboard rate to the camera's own frame rate; an explicitly chosen rate is kept, and the mobile-data rate stays 3s. **(3) Print cards on their own notification channel** (restores the pre-v0.9.27 split): the per-print cards and the all-printers status bar are separate Android categories again, so either can be silenced without the other, and a new **"All printers status bar"** row under Print notifications deep-links to the phone's per-category switches. **(4) No more raw Cloudflare "Bad gateway" page**: a printer whose Pi is still booting behind a live tunnel now gets a plain-English overlay ("the web interface isn't answering yet... Retrying in 5s") that auto-retries until Mainsail loads, in all nine languages. **(5) Two quota-burning runaway clients fixed** (found in the 2026-07-09 Supabase log audit): a printer whose cloud registration is gone (deleted, or moved to another device by a restore) no longer triggers an access request every 4 seconds forever - the app now remembers the "not found" answer and backs off from 1 to 15 minutes; and the **Pi plugin (0.6.13)** no longer re-fetches Moongate's signing keys on every request when the fetch keeps failing (broken crypto install / no network), retrying at most every 2 minutes instead. **Not app-only:** update the Moongate plugin from Moonraker's update panel as well. Also: supabase_flutter 2.15.4. |
| **v0.9.47** | **Brazilian Portuguese + a safer restore.** **(1) New language: Português (Brasil)** - a complete community translation contributed by **@danihmorais** (thank you!), making **nine languages**; pick it on first launch or from the **menu**. | **v0.9.46** | **Quick pause / resume for print notifications.** A new **pause / play button** in the dashboard top bar turns the background **print-notification service** off and on with one tap - handy when your printers will be **off for days** and you don't want Moongate polling them and draining battery in the background (previously you had to force-close the app). **Pausing genuinely stops the background service**, not just the notification, and the paused state is **remembered across app restarts and reboots**, so it stays off until you tap play. The button only appears when **Print notifications** are on (menu → Print notifications), and a **one-time hint** points it out the first time you enable them. iPhone is unaffected (it uses Apple push, not background polling). Localised in all 8 languages. App-only update |
| **v0.9.45** | **Fixes for background wake-up and multi-toolhead preheat.** (1) A printer switched back on while Moongate was in the background could stay stuck on **offline** until you force-closed and reopened it - while the phone freezes the app to save battery, its live-status link to the cloud can drop, so on return it still believed the printer was off even though the notification already showed it back. Moongate now re-checks every printer's live status the instant you return to the foreground. (2) **Multi-toolhead preheat** now sends each hotend its own `SET_HEATER_TEMPERATURE` request instead of one combined command, so setting several tools at once is more reliable, a heater that rejects a target no longer stops the others, and each result is logged for diagnosis. The **preheat panel scrolls** too, so every tool fits on a tool changer with many hotends. App-only update |
| **v0.9.44** | **Multi-toolhead detection fixed at the source (needs the Pi plugin update).** Extra hotends were fetched by the app with separate, transport-fragile requests that could silently fail (e.g. on a remote / VPN connection, or right after the printer booted), leaving a tool changer showing only its first hotend, and v0.9.43's rework hit the same wall because it used the same requests. Now the **Moongate Pi plugin enumerates every `extruder`/`extruderN`, the active `toolhead`, and the `toolchanger`** and folds them into the single authenticated `/status` call the app already reads reliably; the app builds the toolhead list straight from that. **Not app-only:** update the Moongate plugin from Moonraker's update panel as well as the app (plugin 0.6.12). Single-hotend and IDEX printers are handled the same way, nothing changes for them. |
